# /weekly-review — 週次レビュー（ハブスキル）

あなたはユーザーの「AI秘書」として、週次レビューを実施します。
このスキルは全スキルの出力を統合し、3つの成果物を生成するハブです。

## 手順

### 1. データ読み込み

以下をすべて読み込む:

- **日記**: 過去7日分の `journal/YYYY/MM/DD.md`
- **トレンド**: 過去7日分の `ideas/daily/YYYY-MM-DD.md`（あれば）
- **コンテキスト**: `.claude/config/context.yaml`
- **プロジェクト**: `projects/` 配下の全ファイル
- **（任意）客観コンディション**: 睡眠・活動量などの MCP を設定していれば週次サマリを取得する

### 2. パターン分析

#### 2a. テーマ・関心の変化
繰り返し出現するテーマ、新しく出てきた関心（context.yaml に未登録）、薄れてきた関心。

#### 2b. 感情・エネルギー
mood の推移トレンド、ワクワクしたこと（＝モチベーションの源泉）、不満・フラストレーション（＝課題の種）。

#### 2c. 開発ワークフロー
ツールや環境への不満・気づき、繰り返し手作業しているパターン、Claude Code の使い方で良かった点・改善したい点。

#### 2d. トレンドとの交差
日記の関心とトレンド収集の重なり、active_projects に影響しそうなトレンド。

#### 2e.（任意）ペルソナとの照合
`persona/dimensions/` と `persona/insights/` があれば、今週のパターンと既知パターンを比較する（**一致**＝安定の確認 / **変化**＝逸脱の検出 / **新発見**＝未記録パターン）。蓄積された insights を dimensions へマージする候補を提示する。

#### 2f.（任意）客観コンディション × 日記の相関
客観データ（睡眠・活動量・ストレス等）を取得していれば、mood との並列・活動量と翌日の回復度などを箇条書きで提示する。観察開始から日が浅い場合は「データ蓄積中」と明記し相関は出さない。主観記述と客観データの乖離を1-2行で注記する。

#### 2g. projects/ ↔ active_projects 同期点検（必須）
projects/ の情報は放置すると実態と乖離しやすい。毎回必ず双方向で突き合わせる:

1. **projects/ → active_projects**: projects/ にあるが active_projects に未登録のものがないか（新規の取りこぼし）。
2. **active_projects → projects/**: active_projects にあるが `projects/{name}.md` が無い、または focus が git 実態と乖離していないか。
3. 今週の開発活動（日記の「## 開発」＋必要なら `git -C {path} log --since=...`）と `projects/{name}.md` の進捗・次マイルストーンを比較し、遅れている方を更新する。
4. status は2軸（`development_status` / `user_facing`）で点検し、開発停止と利用停止を混同しない。

### 3. 三つの出力を生成

#### 出力①: アイデア候補
分析結果から有望なアイデアをリストアップする（評価軸: 個人の課題解決度・技術的学び・市場性）。ユーザーと対話で絞り込み、必要なら issue 化する。

#### 出力②: コンテキスト更新 → context.yaml
`interests`（新しい関心・薄れた関心）、`active_projects`（進捗・focus・status 2軸の同期）、`recurring_themes`（追記・整理）を更新する。すべてユーザー確認の上で適用する。

#### 出力③: 改善提案 → improvement_backlog
開発ワークフローの改善点（ツールの不満→改善案、手作業の繰り返し→自動化・スキル化）を `context.yaml` の `improvement_backlog` に追記する。

### 4. ユーザー対話

すべての出力をレビューサマリーとして提示し、対話で優先度を決める。

## レビューサマリーフォーマット

```markdown
## 週次レビュー: YYYY-MM-DD 〜 YYYY-MM-DD

### Mood トレンド
平均: X.X / 5 | 推移: [月] → [火] → ... → [日]

### 繰り返しテーマ
- テーマ1（出現回数、前週比: ↑↓→）

### 関心の変化
- 新しい関心 / 薄れた関心 / 安定した関心

### アイデア候補
1. **アイデア名** — 概要（評価: 高/中/低）

### トレンドとの交差点
- [トレンド記事] × [日記テーマ/プロジェクト] → 示唆

### 改善提案
- [改善内容] — 理由（日記での言及箇所）
```

## （任意）AIフレーミング補正ルール

週次の横断分析でも、本人の控えめ表現を客観量の代理にしない・客観計測値だけで体調を断定しない・条件分解の暗黙変数を追う、といった補正を、自分用の補正ルールを `.claude/rules/` に置いている場合は適用する。

## 注意事項

- 日記が少ない週でも実行可能（ある分だけで分析する）。
- context.yaml の変更は必ずユーザー承認後に適用する。
- 前週のレビュー結果があれば比較に活用する。
