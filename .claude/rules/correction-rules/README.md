# 補正ルール — AI に自分を「正しく」読ませる

AI は何も指定しないと、人間を「人間一般」のデフォルト設定で読みます。控えめな人・数値に素直な人・全部を言語化する人——そういう平均値のテンプレで、あなたを上書きしてしまう。日記やペルソナで**材料をどれだけ増やしても、この読み手のズレは直りません**。精度の高い体重計でも目盛りが 2kg ずれていれば、データが増えるほど自信を持って間違えるだけです。

補正ルールは、その「読み手のズレ」を先回りで教えておくファイル群です。「私はこういう人間だ」と説明するのではなく、**「あなた（AI）は私をこう読み違えるので、そこを補正して」**とズレのほうを渡す。`/journal`・`/persona`・`/weekly-review` が、分析に入る前にこれを読みます。

> このディレクトリに入っているのは **型（テンプレート）と汎用の記入例だけ**です。補正ルールの中身は一人ひとりの読み違えられ方に固有なので、雛形をそのまま配れません。自分の対話の中で「ズレ」を採取して、`.claude/rules/` に自分用を足していく前提です。設計の考え方は記事に書きました → [Claude Code に「補正ルール」を渡す](https://zenn.dev/takotyan00/articles/claude-code-correction-rules)。

## 書き方の型（5部構成）

AI がぶれずに従えるよう、いつも同じ骨格で書きます。

1. **中核ルール** — 一文で言い切る。「〜を〜の代わりに使わない」の形が多い。
2. **なぜ必要か** — どの読み違えのクセを、どういう仕組みで踏み抜くか。理由が無いルールは、AI が例外を勝手に作る。
3. **事例** — 実際に踏んだ具体例。抽象論だけだと AI は再現できない。
4. **適用する場面** — どのスキル（`/journal` / `/persona` / `/weekly-review`）の、どのタイミングで効かせるか。
5. **やらないこと** — 過剰補正の禁止。ズレを直すルールはたいてい反対側にも倒れるので、両側にガードレールを引く。

記入例は [`example-understated-expression.md`](./example-understated-expression.md) にあります（汎用化した1本）。

## 「採取」する（机上では作れない）

補正ルールは、頭の中で「こう読み違えられそう」と想像して作るものではありません。**実際に AI が自分を読み違えた瞬間に採取**します。

- 日々の対話や `/weekly-review` で、AI の読みと自分の実感が **ズレた瞬間**を見つける
- そのズレを上の「型」に流し込んで 1 ファイルにする
- 次からは分析の前にそれを読ませる

一度に完璧なルール集を作ろうとしない。ズレを踏むたびに 1 本ずつ増える、育てていく前提の仕組みです。

## 過剰補正への注意（最重要）

5 番の「やらないこと」を必ず書きます。たとえば「控えめ表現を額面で受け取るな」とだけ書くと、AI は逆に何でも「本当はもっと大変だったんですね」と過剰に持ち上げにきます。**補正ルールの目的は「AI に決めつけさせないこと」なので、読み違えを直すために別の決めつけを足したら本末転倒**です。AI がやるのは「問い直す」ところまで。答えを埋めるのは本人。
